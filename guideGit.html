🤔 Git vs GitHub

Deux choses différentes, souvent confondues !

🔧
Git
Un outil installé sur votre ordinateur.

Il enregistre l'historique de vos modifications de code.
🐙
GitHub
Un site web dans le cloud.

Il héberge vos projets Git pour les partager et les sauvegarder en ligne.
🎯 L'analogie parfaite
Git = votre journal intime (sur votre bureau)
GitHub = la bibliothèque où vous déposez votre journal pour le partager

💡 On utilise Git dans le terminal de VSCode,
et on pousse notre code sur GitHub pour le sauvegarder en ligne.

⚡ Les 3 Commandes Essentielles

C'est tout ce dont vous avez besoin pour commencer

git add .
→
Je sélectionne tous mes fichiers modifiés pour les préparer
git commit
→
Je prends une photo de mon code à cet instant avec un message
git push
→
J'envoie cette photo sur GitHub pour la sauvegarder en ligne
Terminal VSCode
# 1. Je prépare mes fichiers
$ git add .

# 2. Je sauvegarde avec un message
$ git commit -m "Ajout de la page d'accueil"

# 3. J'envoie sur GitHub
$ git push
✓ Code envoyé sur GitHub !

📥 Récupérer un Projet : git clone

Télécharger un projet depuis GitHub sur votre ordi

1

Allez sur le projet GitHub

Ouvrez la page du projet sur github.com

2

Copiez l'URL

Cliquez sur le bouton vert "Code" → copiez l'URL HTTPS

3

Clonez dans le terminal

Dans le terminal VSCode, tapez :
git clone [l'URL copiée]

4

Ouvrez le dossier créé

Fichier → Ouvrir un dossier et sélectionnez le dossier téléchargé

Terminal VSCode
$ git clone https://github.com/utilisateur/mon-projet.git
Cloning into 'mon-projet'...
✓ Projet téléchargé !

🆘 Problèmes Fréquents

Solutions rapides aux soucis les plus courants

❌ "git : commande introuvable"
Solution :
→ Git n'est pas installé sur votre machine
→ Allez sur git-scm.com et téléchargez Git, puis relancez VSCode
❌ "Please tell me who you are" (à la première utilisation)
Solution : Git a besoin de savoir qui vous êtes. Tapez :
git config --global user.email "votre@email.com"
git config --global user.name "Votre Nom"
❌ "J'ai oublié d'écrire un message de commit"
Solution :
→ Toujours utiliser git commit -m "votre message"
→ Le message doit décrire ce que vous avez fait : "Ajout du menu", "Correction du bug CSS"…
